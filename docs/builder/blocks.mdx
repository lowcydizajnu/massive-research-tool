A study is an ordered list of **blocks**. Each block is one screen or measure a participant sees. You add, reorder (drag), configure, and remove blocks in the **Builder**.

## Categories

<CardGroup cols={2}>
  <Card title="Questions & judgment">
    Likert scales, multiple choice, open text, accuracy + confidence, sharing intention, constant-sum, drill-down, side-by-side comparison.
  </Card>
  <Card title="Stimuli">
    Text, image, video, and audio stimuli — including social-post skins whose Like / Share / Comment labels you can reword.
  </Card>
  <Card title="Timing">
    Timed exposure (show a stimulus for N seconds) and forced wait.
  </Card>
  <Card title="Image interaction">
    Heat-map, hot-spot (clickable regions), signature, and graphic slider.
  </Card>
  <Card title="Media capture">
    File upload and video/audio recording from the participant.
  </Card>
  <Card title="Flow">
    Embedded data and end-of-study redirect (e.g. back to a recruitment panel).
  </Card>
</CardGroup>

## Configuring a block

Select a block to open its **Configure** panel. Most blocks expose required/optional toggles, answer options, validation, and wording. Wording fields are prefilled with sensible defaults — clear a field to fall back to the default.

## Materials

Reuse images, audio, and video across studies from the **Library → Materials**. Upload once; insert into any stimulus block.

<Tip>
Every block validates as participants go, so a required answer can't be skipped and branching ([conditions](/builder/conditions)) is evaluated live.
</Tip>
