# v1.9–v1.11 — Answer-based flow: branching, conditions, reorder, undo

Built iteratively from live review. Your study is now a real flow you can shape — wire blocks together, gate them on answers, reorder by drag, and step back when you change your mind.

## 🔀 Answer-based branching

Participants take different paths based on what they answer.

- **Wire one block to another** on the Whiteboard. A plain wire just links them; add a condition and it becomes a branch.
- **One block can feed several** (and a block can have several incoming wires) — the path follows whichever condition matches.
- The participant runtime resolves the path **as answers come in** — a gated block only appears once its condition is satisfied, and skipped blocks drop out of the flow and the progress count.

## 🧮 Type-aware condition builder

Select a block → **"Show this block when …"**.

- **Operators match the source block's type** — Likert/slider give `is at least`, `between`, etc.; choice gives `is` / `is any of`; free-text gives `contains`. A stimulus (social post) has no answer, so it can only link, not gate.
- **Combine clauses with AND / OR.**
- Conditions show as labelled wires on the canvas and as **"Shown if …" tags** in the list and Builder.

## ↕️ Drag to reorder

- Grab the **⠿ handle** to reorder blocks in the Builder list and the whiteboard List, with a drop-indicator showing where it'll land.
- Block order is the flow's backbone — a condition can only reference an **earlier** block.

## ✅ Safe edits

- **Reordering that would break a condition** asks first — a clean dialog lists exactly which conditions would be removed (only the ones that move actually affects), so nothing disappears silently.
- **Undo (↶)** in both the Builder and Whiteboard toolbars steps back through your recent block edits (within the session).

## ✏️ Block titles

- Give a block your own **title** (shown everywhere); its module type sits underneath as a quiet `likert-7 · 1.0.0`.

## Under the hood

- A single shared, unit-tested condition engine (`lib/whiteboard/conditions.ts`) powers both the participant runtime and the builder UI, so they can never disagree. Everything lives in the study's blocks JSON — **no database migration**. 200 automated tests green.
