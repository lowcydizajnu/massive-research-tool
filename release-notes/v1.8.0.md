# v1.8.0 — Whiteboard mode + Browse public studies

Two big additions for researchers, shipped together.

## 🗺️ Whiteboard mode

Your study, as a visual graph. The Builder ⇆ Whiteboard toggle is now live.

- **See the structure** — every block is a node; condition-visibility rules are wires from a "Condition: …" entry-point to the blocks they gate. No more hunting through JSON to see what shows when.
- **Edit on the canvas** — add a block (same picker as Builder), click a node to configure it, remove it — all the same edits as Builder, just visual.
- **Compare versions side-by-side** — open any saved version next to your working copy; added blocks are green, removed red, modified amber.
- **It remembers your view** — pan/zoom is saved per study.
- **Accessible List view** — a one-click toggle renders the same blocks as a keyboard-navigable list for screen readers.

## 🔭 Browse public studies

Discover other researchers' work without needing a link.

- **A new Browse destination** — a grid of public, published/preregistered studies.
- **Filter + sort** — by tag (intersect multiple), by author, and by most-recent or most-replicated.
- **Read before you borrow** — open any study's read-only Details (its blocks, author, version).
- **Replicate in one click** — fork a public study into your workspace and land straight in the Builder; the original author sees the replication in their Activity.
- **Follow** authors and tags right from the cards.

## Under the hood

- Canvas built on React Flow (MIT) as a translation layer over the existing block model — no new data model (ADR-0020).
- Browse runs over existing data (public visibility + published/preregistered versions); cursor-paginated (ADR-0018).
- 177 automated tests; typecheck, lint, and build green.

## Not yet (coming later)

Free-form sketching on the canvas, sketch-to-block, real-time canvas collaboration, full-text Browse search, and a "save this search" follow — all deferred to a future release.
