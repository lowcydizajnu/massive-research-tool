# 03_design/inspiration — what we're learning from other tools

This folder captures inspiration that shapes our design language. Per `00_meta/rules/design-rules.md`, inspiration is annotated, not dumped. Each batch is a markdown file with what we looked at, what we liked specifically, and what we'd avoid.

## How to add inspiration

- **One batch file per session**, named `YYYY-MM-DD-{short-slug}.md`.
- For each reference, capture: source / annotated screenshot reference / **what we like specifically** / **what we'd avoid** / pattern to lift.
- Be specific. "Modern UX" doesn't help; "the way TheyDo's right-drawer slides in with a focus on the selected row" does.
- Anti-references are as useful as positive ones — note "don't do this and why."

## How synthesis works

Inspiration batches feed into `03_design/design-language-brief.md` — the single document that articulates our visual identity, layout system, and interaction patterns. The brief is the source of truth; batches are the evidence that informed it.

When the brief changes, link to the inspiration batch entries that drove the change so future contributors can trace the reasoning.

## Categories of inspiration we care about

- **Tooling adjacent to ours** — research, productivity, complex SaaS (Linear, Figma, Notion).
- **Form/survey builders** — direct competitors (Typeform, Maze, Qualtrics — anti-reference).
- **Whiteboard / node-graph tools** — for our second creation mode (Figma, Miro, n8n).
- **Service-design / research tools** — TheyDo, Dovetail, Optimal Workshop.
- **Color/type philosophy** — Stripe, Linear, GitHub.
- **Empty states, errors, loading** — wherever character lives.
- **Anti-references** — Qualtrics, classic OSF UI, legacy SPSS UIs. "Don't do this" is as important as "do that."
